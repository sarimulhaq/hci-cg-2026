Norman’s Action Cycle

Norman’s Action Cycle explains how users interact with systems step by step: forming a goal, planning an action, executing it, and evaluating the result (Dix et al., Ch. 9, p. 437).

Key challenges are the gulf of execution—difficulty translating intention into action—and the gulf of evaluation—difficulty understanding feedback. Effective interfaces reduce both gaps.

This model highlights user cognition and feedback, helping designers spot usability problems.

GOMS Model

GOMS stands for Goals, Operators, Methods, and Selection rules. It is a structured way to describe how expert users perform tasks (Dix et al., Ch. 12, p. 437).

Goals are what users want to achieve. Operators are basic actions like clicks or keystrokes. Methods are sequences of operators to achieve goals. Selection rules determine which method to use in a situation.

GOMS helps break down complex tasks into predictable steps, making it easier to analyze efficiency.

Keystroke-Level Model (KLM)

KLM is a detailed extension of GOMS that predicts how long an expert user takes to perform a task without errors (Dix et al., Ch. 12, p. 437).

Tasks are divided into low-level operators: keystrokes, pointing, mental preparation, and system response time. Summing these gives the predicted execution time.

KLM focuses solely on efficiency, not decision-making or interpretation.

Model Comparison

Norman vs GOMS vs KLM
Model Focus Design Insight
Norman User thinking and feedback Reduces execution and evaluation gaps
GOMS Task structure and efficiency Breaks tasks into predictable steps
KLM Execution time prediction Estimates expert user performance speed

Together, these models provide complementary perspectives: Norman explains user thinking, GOMS structures tasks, and KLM predicts performance timing.
